Installation Steps

1

Clone the Repository

Download the configuration files to your local machine:

git clone https://github.com/wkcollis1-eng/home-assistant-config.git
cd home-assistant-config

Download the ZIP file from GitHub:

wget https://github.com/wkcollis1-eng/home-assistant-config/archive/refs/heads/master.zip
unzip master.zip
cd home-assistant-config-master
2

Configure Secrets

Create your secrets file with API keys and sensitive information:

# Copy the example file
cp secrets.yaml.example secrets.yaml

# Edit with your credentials
nano secrets.yaml

Required secrets:

# Pirate Weather API Key (or your weather service)
pirate_weather_api_key: "your_api_key_here"

# Home Location
home_latitude: 41.28
home_longitude: -72.81
home_elevation: 50

# Optional: Notification services
notify_email: "your-email@example.com"
üîë Getting API Keys

Pirate Weather: Sign up at pirateweather.net for a free API key (500 calls/day free tier)

3

Update Entity IDs

Customize the configuration to match your thermostat entity IDs:

Find Your Entity IDs

In Home Assistant:

  1. Go to Developer Tools ‚Üí States
  2. Search for your thermostat entities
  3. Note the entity IDs (e.g., climate.first_floor, climate.second_floor)

Update Configuration

Edit configuration.yaml and replace the example entity IDs:

# Find and replace these entity IDs with yours:
# Old: climate.1f_thermostat
# New: climate.your_first_floor

# Old: climate.2f_thermostat  
# New: climate.your_second_floor

# Search for all instances in configuration.yaml
‚ö†Ô∏è Important

Make sure to update ALL occurrences of the entity IDs throughout the configuration file. Use find-and-replace in your text editor to ensure you don't miss any.

üìä Establishing Your Baselines

The configuration includes baseline values from a Connecticut home. You can start with these and they'll auto-adjust over time, or calculate your own using the Baseline Analysis methodology.

5

Install Custom Components

Install required custom integrations via HACS (Home Assistant Community Store):

Install HACS (if not already installed)

  1. Follow instructions at hacs.xyz
  2. Restart Home Assistant after HACS installation

Install Pirate Weather

  1. Open HACS in Home Assistant
  2. Click Integrations
  3. Click the + button
  4. Search for "Pirate Weather"
  5. Click Download
  6. Restart Home Assistant

Configure Pirate Weather

# Already included in configuration.yaml
weather:
  - platform: pirateweather
    api_key: !secret pirate_weather_api_key
    latitude: !secret home_latitude
    longitude: !secret home_longitude
    mode: hourly
6

Copy Files to Home Assistant

Copy the configuration files to your Home Assistant config directory:

  1. Connect to your Home Assistant Samba share (usually \\homeassistant.local\config)
  2. Copy these files/folders to the config directory:
    • configuration.yaml
    • automations.yaml
    • scripts.yaml
    • climate_daily_norms.csv
    • scripts/ folder
    • dashboards/ folder
  3. Merge or replace existing files as needed
# SSH into your Home Assistant instance
ssh root@homeassistant.local

# Navigate to config directory
cd /config

# Copy files (if you have them on a USB or network location)
# Or use SCP from your computer:
# scp -r /path/to/downloaded/config/* root@homeassistant.local:/config/
‚ö†Ô∏è Backup First!

Before copying files, backup your existing configuration.yaml and other files. You may want to merge this configuration with your existing setup rather than replacing entirely.

7

Validate Configuration

Check your configuration for errors before restarting:

Using Home Assistant UI

  1. Go to Developer Tools ‚Üí YAML
  2. Click Check Configuration
  3. Fix any errors reported

Using Command Line (Advanced)

# SSH into Home Assistant
ssh root@homeassistant.local

# Run configuration check
ha core check

# If using Docker:
docker exec homeassistant python -m homeassistant --script check_config --config /config
‚úÖ Configuration Valid

If validation passes, you're ready to restart Home Assistant!

8

Restart Home Assistant

Restart to load the new configuration:

  1. Go to Settings ‚Üí System ‚Üí Restart
  2. Click Restart Home Assistant
  3. Wait for the system to come back online (usually 1-2 minutes)

After restart, verify the sensors are working:

  1. Go to Developer Tools ‚Üí States
  2. Search for sensor.hdd_ to see HDD sensors
  3. Search for sensor.1f_runtime and sensor.2f_runtime
  4. Verify values are populating
9

Import Dashboards

Add the HVAC monitoring dashboards to your Home Assistant:

  1. Go to Settings ‚Üí Dashboards
  2. Click + Add Dashboard
  3. Choose New dashboard from scratch
  4. Name it "HVAC Monitoring"
  5. Click the ‚ãÆ menu ‚Üí Edit Dashboard
  6. Click the ‚ãÆ menu again ‚Üí Raw configuration editor
  7. Copy the YAML from dashboards/cards/ folder
  8. Paste and save
üìä Dashboard Cards

The repository includes pre-built dashboard cards in the dashboards/cards/ folder. Start with these and customize to your preferences.

Troubleshooting

Common Issues

‚ùå Sensors Not Showing Values

Cause: Entity IDs don't match your thermostats

Solution: Double-check entity IDs in configuration.yaml match your actual thermostat entities

‚ùå HDD Values Are Zero

Cause: Weather integration not providing temperature

Solution: Verify Pirate Weather integration is configured and providing sensor.pirateweather_temperature

‚ùå Configuration Check Fails

Cause: YAML syntax errors or missing integrations

Solution: Check the error message, verify YAML indentation, ensure all custom components are installed

‚ùå Automations Not Triggering

Cause: Time-based automations may need initial data

Solution: Wait 24 hours for daily automations to collect initial data, or manually trigger via Developer Tools
